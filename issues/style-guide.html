# data-middle-platform 风格指南

Tip

本项目的风格指南主要是参照 vue 官方的风格指南。大部分规则也都配置在了eslint-plugin-vue之中，当没有遵循规则的时候会报错，详细内容见 eslint 配置。

# Component

所有的Component文件都是以大写开头 (PascalCase)，这也是官方推荐，除了index.vue

例子：

  • @/components/Pagination/index.vue
  • @/views/example/components/Button.vue

# JS 文件

所有的.js文件都遵循横线连接 (kebab-case)。

例子：

  • @/utils/scroll-to.js
  • @/utils/get-page-title.js

# Views

在views文件下，代表路由的.vue文件都使用横线连接 (kebab-case)，代表路由的文件夹也是使用同样的规则。

例子：

  • @/views/error-page/index.vue
  • @/views/data/open-api.vue

使用横线连接 (kebab-case)来命名views主要是出于以下几个考虑。

  • 横线连接 (kebab-case) 也是官方推荐的命名规范之一
  • views下的.vue文件代表的是一个路由，所以它需要和component进行区分(component 都是大写开头)
  • 页面的url 也都是横线连接的，比如https://localhost/data/open-api，所以路由对应的view应该要保持统一
  • 没有大小写敏感问题
